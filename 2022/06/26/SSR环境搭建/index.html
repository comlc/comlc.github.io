SSR环境搭建
2022-06-26|其他SSR环境|
字数总计: 790|阅读时长: 4 分钟

简介

Shadowsocks-R服务节点

服务器环境

CentOS-7.9-x86_64

安装SSR

  1. 获取SS四合一脚本（四个分别是 SS-Python，SSR，SS-Go，SS-libev，我们选择的是SSR）

    wget --no-check-certificate -O shadowsocks-all.sh https://raw.githubusercontent.com/teddysun/shadowsocks_install/master/shadowsocks-all.sh

    chmod +x ./shadowsocks-all.sh

  2. 执行脚本./shadowsocks-all.sh，会提示Which Shadowsocks server you'd select，选择ShadowsocksR

  3. 下一步提示Please enter password for SSR，输入密码
  4. 下一步提示Please enter a port for SSR [1-65535]，输入服务端口号
  5. 下一步提示Please select stream cipher for SSR，选择加密方式none，下面的协议会自带加密
  6. 下一步提示Please select protocol for SSR，选择协议auth_chain_a
  7. 最后提示Please select obfs for SSR，选择tls1.2_ticket_auth
  8. 等待安装完成即可，备注（以上加密协议亦可选择aes-256-cfb，origin，plain）

开启BBR加速

BBR是Google开发的TCP拥塞算法，BBR模块需要Linux4.9及以上的内核支持，CentOS7的发布版标配的是kernel-3.10，所以首先需要升级内核到大于等于4.9的版本

  1. 查看当前系统内核版本：uname -r

  2. 安装eprl的源：

    sudo rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org

    若执行报错，则先更新yum update

    sudo rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-2.el7.elrepo.noarch.rpm

  3. 安装最新的内核版本：sudo yum --enablerepo=elrepo-kernel install kernel-ml -y

  4. 查看系统现在所有的内核：rpm -qa | grep kernel

  5. 安装的最新内核是kernel-ml-6.0.3-1.el7.elrepo.x86_64，需修改grub2的启动项

    查看默认启动内核：grub2-editenv list

    查看所有内核：cat /boot/grub2/grub.cfg | grep menuentry

    修改默认启动内核：grub2-set-default 'CentOS Linux (6.0.3-1.el7.elrepo.x86_64) 7 (Core)'

    重启服务器：reboot

  6. 开启BBR加速：sudo wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh && chmod +x bbr.sh && ./bbr.sh

    或执行以下命令直接开启：

  7. 验证BBR：执行lsmod | grep bbr或sysctl -n net.ipv4.tcp_congestion_control均有BBR字样即表示启动成功

开启防火墙

  1. 安装防火墙并启动：yum install firewalld firewall-config

    启动：systemctl start firewalld

    查看状态：systemctl status firewalld

    设置开机自启动：systemctl enable firewalld

  2. 开放SSR等部分端口：

    查看防火墙及开启的端口firewall-cmd --list-all

    永久添加端口firewall-cmd --zone=public --add-port=8080/tcp --permanent

    使配置生效firewall-cmd --reload
